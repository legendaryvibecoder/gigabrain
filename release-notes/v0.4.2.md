# Gigabrain 0.4.2

`0.4.2` is the onboarding and packaging follow-up to the `0.4` rollout.

## What changed

- the published package now includes `npm run setup`
- the setup wizard enables the Obsidian surface by default and builds the first vault immediately
- the setup wizard refreshes existing `AGENTS.md` memory instructions instead of only appending once
- the package now includes `vault:report`
- installation and onboarding docs explain the Obsidian-first `v0.4` flow more clearly

## Why this matters

The big `0.4` product direction was already there, but the install flow still had rough edges:

- users could read about `npm run setup` before it actually existed in the package
- the Obsidian surface was important but not explained strongly enough during onboarding
- fresh installs needed clearer expectations that a new vault can look sparse until memories exist

`0.4.2` closes those gaps so the shipped npm package matches the intended experience.

## Recommended setup flow

1. `npm install @legendaryvibecoder/gigabrain`
2. `npm run setup -- --workspace /path/to/workspace`
3. Open `<workspace>/obsidian-vault/Gigabrain` in Obsidian
4. Start at `00 Home/Home.md`
5. Use `vault pull` to sync the generated surface to another machine
