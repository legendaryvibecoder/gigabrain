# Gigabrain v0.7.0 Cross-Memory Pivot

Released: 2026-04-24

## Summary

`v0.7.0` repositions Gigabrain as the local-first Memory Passport and control plane for agents. The product face is Passport + Auditor + Handoff Layer; the cross-memory bus for Codex, Claude Code, OpenClaw, Hermes-style bridges, Cursor/Windsurf, and explicit manual imports from closed cloud products is the architecture underneath.

Native memories are now good enough to be useful, but they are still usually product-owned and siloed. Gigabrain does not replace those memories. It connects local surfaces where the host exposes files, records provenance, deduplicates equivalent memories, redacts obvious secrets, and exports safe host-specific briefs.

## Highlights

- **Memory Passport**
  - `gigabrainctl passport` writes `memory-passport.md`, `memory-passport.html`, `memory-passport.json`, and handoff briefs
  - the Passport includes source inventory, host readiness, readiness verdict, dedupe audit, contradiction audit, stale-memory audit, provenance gaps, and secret-risk flags
  - handoff briefs are generated for `AGENTS.md`, `CLAUDE.md`, ChatGPT, Claude.ai, Gemini, and Microsoft Copilot manual paste/import
  - secret-risk memories are omitted from handoff/export briefs entirely; the audit still reports them with redacted previews
- **Host memory sync**
  - `gigabrainctl sync-hosts` indexes local Codex memories, Claude Code memory folders, Cursor/Windsurf rules, and OpenClaw native memory where visible
  - Hermes Agent gets first-class MCP setup through `gigabrain-hermes-setup` and read-only sync for `~/.hermes/memories`
  - Legacy OpenClaw/Gigabrain `registry.sqlite` backups can be imported with `gigabrainctl import-openclaw` while retaining source provenance and evidence snippets
  - manual cloud imports are explicit and tagged as `manual_import`
  - ChatGPT, Claude.ai, Gemini, and Microsoft Copilot imports are marked `bidirectional_disallowed`
- **Cross-agent provenance**
  - imported memories carry `source_host`, `source_kind`, `source_path`, `source_line`, and `sync_policy`
  - exact duplicates collapse into one current memory while retaining all host provenance links
- **MCP/API focus**
  - new `gigabrain_sources` tool reports source counts and freshness
  - new `gigabrain_sync_status` tool reports host-by-host sync diagnostics
  - new `gigabrain_export_brief` tool produces safe AGENTS.md, CLAUDE.md, or manual cloud-product briefs
- **Positioning**
  - README and docs now describe Gigabrain as a local-first Memory Passport/control plane
  - docs explicitly state that closed cloud memories require user-controlled import/export and are not scraped

## Upgrade Notes

- Existing stores continue to work without migration. New source metadata columns default to `gigabrain`, `registry`, and `read_only`.
- Existing MCP tools remain stable: `gigabrain_recall`, `gigabrain_remember`, `gigabrain_checkpoint`, `gigabrain_provenance`, `gigabrain_recent`, and `gigabrain_doctor`.
- `gigabrainctl sync-hosts` is additive. It does not mutate native host memory files.
