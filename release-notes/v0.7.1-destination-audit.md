# Gigabrain v0.7.1 Destination Audit

v0.7.1 is a release-hardening update for the cross-memory pivot. It audits each supported destination, tightens MCP metadata, and documents the current "OnePassword for Memory" status.

## Highlights

- Added a destination audit matrix for OpenClaw, Hermes/Nimbus, Codex, Claude Code/Desktop, MCP clients, Cursor/Windsurf, manual cloud imports, and Memory Passport.
- Updated MCP server version reporting so the MCP server advertises the package version.
- Added MCP tool annotations for read-only and additive-write tools.
- Strengthened MCP integration coverage so all tools validate structured output and serialized JSON text fallback.
- Documented that npm remains the release package manager for v0.7.1; pnpm is deferred until there is a workspace or monorepo reason.

## Product Status

Gigabrain is now a Memory Passport and memory control-plane MVP. It already provides inventory, provenance, dedupe, recall, host sync, audit reports, safe handoff briefs, OpenClaw import, Hermes bridge, Codex/Claude setup, and MCP tools.

Vault-grade "OnePassword for Memory" still needs encrypted portable bundles, signed export manifests, delete/tombstone workflows, review/approval UI, trust labels, and stronger secret governance.

## Validation

Release validation should include the full test suite, high-severity npm audit gate, npm pack dry-run, npm publish dry-run, Hermes MCP smoke, and packaged install entrypoint smoke.
