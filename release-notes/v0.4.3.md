# Gigabrain v0.4.3

## Summary

`v0.4.3` is a recall hygiene patch release. It makes Gigabrain's automatic recall cleaner in live chats by removing internal provenance from the injected context, filtering persisted transcript/recall artifacts out of native memory recall, and making stale relative date wording safer.

## What's fixed

- Hidden Gigabrain recall injection no longer includes visible provenance like `src=...`, file paths, or memory ids.
- Native recall skips stored artifacts such as `<gigabrain-context>`, `query:`, `fallback:`, `Source: ...`, and transcript-style `user:` / `assistant:` lines when indexing and querying daily notes.
- Older memories that say `today`, `heute`, or similar relative terms are injected as recorded historical notes, for example:
  - `Recorded on 2026-02-23; any relative dates in this memory refer to that date.`

## Why this matters

- User-facing answers are less likely to leak internal memory mechanics.
- Session logs no longer feed old recall scaffolding back into the memory system.
- Time-sensitive memories are less likely to sound current when they were actually recorded days or weeks earlier.

## Operational note

Gigabrain now keeps its own automatic recall context clean, but OpenClaw's separate `memory_search` / `memory_get` tools still have their own citation setting. If you want direct-chat replies without visible `Source:` lines when those tools are used, set:

```json
{
  "memory": {
    "citations": "off"
  }
}
```

That OpenClaw runtime setting is not bundled by the Gigabrain npm package, but it is recommended when running both systems together.
