# Gigabrain 0.4

Gigabrain `0.4` turns memory into a dual surface product:

- the runtime workspace and registry remain the source of truth
- Obsidian becomes the best human-readable view of that memory
- the web console becomes the operational companion

## Highlights

### Obsidian Memory Surface

Gigabrain can now build a structured Obsidian vault with:

- `00 Home` for the landing page and health summary
- `10 Native` for `MEMORY.md` and daily/session notes
- `20 Nodes/active` for one note per active registry memory
- `30 Views` for Active Memories, Relationships, Review Queue, Recent Archives, Native Sources, Promoted Memories, and Registry-only Memories
- `40 Reports` for manifest, freshness, and latest maintenance artifacts

This gives memory a clear visual home instead of leaving everything trapped in the registry.

### Hybrid Memory Model

`0.4` restores native markdown as a first-class layer while keeping the structured Gigabrain registry:

- explicit remember intent can write to native markdown and the registry together
- durable native notes can be promoted back into structured memory with provenance
- normal chat stays conservative and does not silently create durable memory

The goal is simple: memory should feel natural to use, not like users need to learn internal XML.

### Better Recall and Memory Hygiene

The memory system is now more resilient and more useful in practice:

- hybrid dedupe catches malformed near-duplicates better
- plausibility heuristics reduce noisy low-value captures
- relationship and durable-personal facts are treated more carefully
- nightly maintenance now produces clearer artifacts and ends with a vault build

### Better Local Model Routing

Local Qwen usage is now task-specific instead of one-size-fits-all:

- structured review tasks use stable low-temperature profiles
- general chat can stay closer to the model defaults
- this improves JSON reliability without forcing greedy decoding

### Web Console Surface View

The FastAPI console now understands the new surface too:

- shared surface summary
- vault freshness
- native vs registry counts
- review queue and recent archives at a glance

## What users should know

- Obsidian is strongly recommended for `0.4` if you want the full memory experience.
- Gigabrain still works without Obsidian for capture and recall.
- A fresh vault can look sparse at first. That is expected until native notes or remembered facts exist.
- The setup wizard now enables the surface by default and points users to `00 Home/Home.md`.

## Setup recommendation

Recommended first-run flow:

1. Install Gigabrain
2. Run `npm run setup -- --workspace /path/to/workspace`
3. Open `<workspace>/obsidian-vault/Gigabrain` in Obsidian
4. Start at `00 Home/Home.md`
5. Use `vault pull` to sync the generated surface to another machine
