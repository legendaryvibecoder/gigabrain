# Gigabrain v0.6.1

Released: 2026-03-18

## Summary

`v0.6.1` is a small post-release hardening update on top of `v0.6.0`.

It focuses on three things that matter for a stable install base: fresh standalone setup reliability, safe upgrades for older native stores, and stricter scoped access behavior in recall and memory surfaces.

## Availability

- npm: `npm install @legendaryvibecoder/gigabrain`
- GitHub release: `v0.6.1`
- Claude Desktop asset: `gigabrain-claude-desktop-0.6.1.dxt`

## Highlights

- **Fresh-install robustness**
  - generated Codex and Claude helper scripts no longer embed ephemeral `~/.npm/_npx/...` cache paths
  - durable fallback now resolves through `npx --yes --package @legendaryvibecoder/gigabrain@0.6.1 ...`
  - setup now fails closed on malformed standalone configs instead of silently overwriting them
  - `gigabrainctl doctor --config ...` now reports missing explicit configs clearly before SQLite startup
- **Legacy-store safety**
  - older `memory_native_chunks` tables are now migrated before scope-based indexes are created
  - legacy Nimbus and OpenClaw registries no longer fail on startup with `no such column: scope`
- **Deep-audit hardening**
  - scoped world-model/entity lookups now fail closed instead of surfacing data from unauthorized scopes
  - multi-entity reranking now uses real entity signals instead of ineffective opaque-id text matching
  - partial-cache semantic rerank no longer penalizes uncached rows when embeddings are missing
  - memory API relation responses now filter related rows by the same scope/access rules as their anchor memories

## Upgrade Notes

- `v0.6.0` users should rerun `gigabrain-codex-setup` or `gigabrain-claude-setup` once if they want refreshed helper scripts with the durable package-spec fallback.
- Existing legacy installs under `~/.codex/gigabrain/` remain supported and are reused in place.
- No MCP or HTTP API surface changes are required for clients upgrading from `v0.6.0`.

## Validation

Release validation for `v0.6.1` included:

- `npm test`
- `npm pack --dry-run`
- `npm run test:release-live`
- `npm run eval:deep-recall`
- fresh packaged setup flows for Codex and Claude without local `node_modules`
- legacy native-store migration verification on Nimbus
- Claude Desktop bundle build verification with the hardened launcher
